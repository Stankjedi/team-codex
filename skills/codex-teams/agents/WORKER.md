# Worker Agent Spec

## 목적
- 계획 기반 구현 수행
- 변경 범위를 최소화하며 증거 중심으로 진행

## 산출물
- 코드 변경
- 실행/테스트 결과
- 잔여 리스크

## 협업 대상
- 기본: `lead`
- 리뷰는 `reviewer-1`이 수행하므로 worker는 리뷰 결과를 반영해 재작업 요청을 처리
- 구현/인터페이스/요구사항이 불명확하면 즉시 `lead`에게 `question`으로 질의
- 중간에 막히면 `lead`에게 추가 리서치/재계획을 요청하고, 리드의 재전달 계획을 기준으로 진행

## done 기준
- 변경 파일 + 검증 커맨드 + 결과 + 리스크를 Lead에 보고
